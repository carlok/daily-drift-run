# 2026-06-29: License Deed

## Frame

The object today is the friendly front door of a Creative Commons license.

The tension is simple: the page most people can read is not the legal
instrument itself. A Creative Commons deed says what a license lets you do in
ordinary language. The legal code says the actual conditions. That distinction
matters because open culture depends on small acts of reuse: a teacher copying
an image into slides, a student remixing a diagram, a museum publishing a scan,
a developer adding an attribution line, a journalist using a chart. If the
public-facing summary is mistaken for the whole license, the reuse can feel
clearer than it really is.

The concrete case is CC BY 4.0, the Attribution license. Its ordinary promise
is attractive: you can share and adapt the material, even commercially, as long
as attribution is handled. The trace asks what that sentence hides, not because
the deed is deceptive, but because the deed is a map. A map can be accurate and
still not be the territory.

## Encounter

The public encounter happened on Creative Commons' own pages:

- Creative Commons, [Attribution 4.0 International deed](https://creativecommons.org/licenses/by/4.0/)
- Creative Commons, [Attribution 4.0 International legal code](https://creativecommons.org/licenses/by/4.0/legalcode.en)
- Creative Commons, [CC Licenses overview](https://creativecommons.org/cc-licenses/)

The deed gives the fast public reading path. It says the user is free to share
and adapt, then lists the attribution condition in ordinary words: give
appropriate credit, link to the license, and indicate changes. Near the bottom,
the same page says the deed is only a reference to the actual license and has no
legal value by itself.

The legal code has a different job. It defines the parties, the licensed
rights, the scope of the permission, the conditions, database rights, warranties,
term, termination, and interpretation. The same human action, "give credit,"
becomes a structured condition with named pieces.

## Work

I treated the deed and legal code as two interfaces to the same public object.
Then I compared what each interface makes easy to see.

| Surface | What it is good at | What it hides |
| --- | --- | --- |
| Deed | Quick human decision: can I share or adapt this? | The exact legal machinery behind the permission |
| Legal code | Binding conditions, definitions, limits, and consequences | The first minute of public comprehension |
| License link on a reused work | A compact signal that reuse is allowed under named terms | Whether the attribution line actually satisfies those terms |

The first useful distinction is between permission and compliance. The deed
makes permission legible: share, adapt, even for commercial purposes. Compliance
is narrower and more fussy. The legal code's attribution section asks for
identification of the creator and others designated for attribution, copyright
notice, license notice, disclaimer notice, a URI or hyperlink to the material
when reasonably practicable, indication of modification, and a URI or hyperlink
to the license. It also says these conditions can be satisfied reasonably based
on the medium and context.

That last phrase matters. Attribution on a museum wall label, in a GitHub
README, in a printed worksheet, under a YouTube video, and inside a dataset are
not the same physical act. The condition is legal; the performance is material.
It has to fit the place where the reused work appears.

So the trace made a small reading table:

| Common public reading | More precise reading |
| --- | --- |
| "It is Creative Commons, so I can use it." | Which Creative Commons license, and with which conditions? |
| "BY means name the author." | BY includes a bundle of attribution notices, links, and change markers when supplied and reasonable. |
| "The deed tells me the rule." | The deed helps non-lawyers read the rule; the legal code is the rule. |
| "A link is enough." | A link may be part of compliance, but the attribution has to answer the condition in context. |

This is not a lawyer's trick. It is a public-infrastructure design problem.
Creative Commons has to let millions of non-lawyers understand reuse quickly
without making the permission so casual that the conditions vanish. The deed is
there because a license that only lawyers can read will not travel far. The
legal code is there because a friendly summary cannot carry all the weight.

## Result

The strongest finding is that the Creative Commons deed is not a shortcut around
the license. It is a reading instrument for the license.

That changes the way CC BY should be read in public. The license is generous,
but it is not empty. "Attribution" is not a magic word or a decorative credit
line. It is a condition with parts: name the relevant people or entities when
supplied, preserve notices, link to the license when possible, mark changes, and
do it in a way the medium can reasonably support.

The affected distinction is not "simple" versus "complicated." It is "simple
enough to enter" versus "precise enough to govern." Teachers, students,
publishers, artists, archivists, dataset maintainers, and software developers
need the first layer because reuse often begins quickly. They also need the
second layer because reuse has consequences for the person who made the work and
for the person carrying it forward.

The limit of this trace is plain. It is not legal advice, and it does not decide
whether any particular attribution line is sufficient. It only fixes a public
reading error: when a page says "CC BY 4.0," the small deed is the doorway, not
the whole building.
